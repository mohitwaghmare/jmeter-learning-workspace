# Apache JMeter - CI/CD Integration

## What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment (or Continuous Delivery).

It automates building, testing, and deploying applications.

---

## Why Integrate JMeter?

* Automate performance testing
* Detect performance regressions
* Generate reports automatically
* Improve software quality
* Reduce manual effort

---

## Common CI/CD Tools

* Jenkins
* GitHub Actions
* Azure DevOps
* GitLab CI/CD

---

## Typical Workflow

1. Developer pushes code.
2. CI/CD pipeline starts.
3. Application is built.
4. JMeter tests execute.
5. HTML Dashboard Report is generated.
6. Results are published.

---

## Best Practices

* Execute JMeter in Non-GUI mode.
* Store reports as build artifacts.
* Fail the pipeline when performance thresholds are exceeded.
* Keep test plans in source control.

---

## Interview Questions

### What is CI/CD?

CI/CD automates software build, testing, and deployment processes.

### Why integrate JMeter into CI/CD?

To automatically execute performance tests after every build or deployment and detect performance issues early.

### Which CI/CD tools support JMeter?

* Jenkins
* GitHub Actions
* Azure DevOps
* GitLab CI/CD

---

## Summary

CI/CD integration enables automated performance testing, helping teams identify performance issues early and maintain application quality.
