# Apache JMeter - HTML Dashboard Reports

## What is an HTML Dashboard Report?

An HTML Dashboard Report is a graphical performance report generated by Apache JMeter from a `.jtl` results file.

It provides an easy-to-understand view of test execution and performance metrics.

---

## Key Metrics

* Response Time
* Throughput
* Error Percentage
* Active Threads
* Response Time Percentiles (90%, 95%, 99%)
* Requests per Second

---

## Benefits

* Professional performance reports
* Interactive charts and graphs
* Easy analysis of performance metrics
* Shareable with stakeholders
* Helps identify performance bottlenecks

---

## Best Practices

* Execute performance tests in Non-GUI mode.
* Generate reports after test completion.
* Archive reports with each performance test.
* Compare reports across multiple executions.
* Avoid using GUI mode for large load tests.

---

## Interview Questions

### What is an HTML Dashboard Report?

An HTML Dashboard Report is a graphical report generated from a JMeter `.jtl` results file that provides performance metrics and charts.

---

### Which file is required to generate the report?

The `.jtl` results file.

---

### Why is Non-GUI mode recommended?

Non-GUI mode consumes less CPU and memory, making it ideal for load and performance testing.

---

## Summary

HTML Dashboard Reports provide a professional and visual way to analyze performance test results. They help engineers understand response times, throughput, errors, and system behavior during testing.
