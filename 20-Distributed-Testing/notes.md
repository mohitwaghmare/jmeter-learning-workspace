# Apache JMeter - Distributed Testing

## What is Distributed Testing?

Distributed Testing allows multiple JMeter machines to generate load together.

One machine acts as the Master and controls one or more Worker machines.

---

## Architecture

Master

↓

Worker 1

↓

Worker 2

↓

Worker 3

↓

Target Application

---

## Why Use Distributed Testing?

Large-scale applications may require thousands of concurrent users.

A single machine may not have enough CPU or memory to generate that load.

Distributed Testing spreads the workload across multiple systems.

---

## Requirements

- Same Java Version
- Same JMeter Version
- Network Connectivity
- Open RMI Ports
- Same Test Plan

---

## Advantages

- Supports high user loads
- Better scalability
- Efficient resource utilization
- Enterprise-ready testing

---

## Best Practices

- Execute tests in Non-GUI mode.
- Monitor all Worker machines.
- Keep software versions consistent.
- Use reliable network connectivity.

---

## Interview Questions

### What is Distributed Testing?

Distributed Testing uses multiple JMeter machines to generate load simultaneously.

---

### Why use Distributed Testing?

To simulate large numbers of virtual users that cannot be generated by a single machine.

---

### What is the role of the Master?

The Master controls the execution and collects results from Worker machines.

---

### What is the role of Workers?

Workers execute the test plan and generate the load.

---

## Summary

Distributed Testing enables large-scale performance testing by distributing virtual users across multiple JMeter machines.
